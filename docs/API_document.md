# ユーザーの動き
1. ログイン
2. 必ず寄りたい場所を入力
3. 興味あるタグを入力
4. 自然言語でより詳しい興味内容を入力
5. リコメンドボタンを押す
6. 一覧から行きたい場所を選択し，追加ボタンを押す
7. それらを含んだ最終的な工程表を得る

# 必要な機能
- POST FukuokaAI-test/recommend リコメンド機能
入力：寄りたい場所（リスト），興味タグ（リスト）
出力：場所（リスト）

処理の流れ：
1. 出発地点とゴール地点を指定したのち，それ以外の必ず寄りたい場所の座標を得る
2. 出発地点とゴール地点を含む全ての座標で，それぞれ一番距離が近い組み合わせを作り，全ての点が線でつながるようにする（独立した枝ができた場合，枝同士で一番近い地点同士を結ぶ）
3. 全ての枝で，半径が　枝の長さ/√3　となる円内で，興味タグで検索をnearby search APIで検索する 
4. その結果を一枚の写真と共にレビューの高い順に合計10件表示する

- POST FukuokaAI-test/add/{place_id} リコメンド場所をリストに追加
入力：なし
出力：なし

処理の流れ：
1. ボタンを押した際に，ボタンの場所idを取得
2. 行きたい場所リストにそのpleace_idを追加

- POST FukuokaAI-test/result リコメンド追加してルートを提案
入力：場所（リスト）
出力：場所（リスト）

処理の流れ：
1. 行きたい場所リストを取得
2. そのリストをRouteAPIで送信
3. 最適化されたルート情報を取得
